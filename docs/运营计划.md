# 淘宝选品可视化系统运营计划

## 已实行的系统
- 可视化网站：`index.html` 展示10个候选品、图片、三平台官方预估到手价、30天销量、累计评价、建议价格、成本上限、风险提示和综合热度分。
- 商品池：`data/product_catalog.json` 维护单独SKU、平台商品ID、精确规格匹配词和打分输入，不保存推测价格区间。
- 市场缓存：`data/market_data.json` 只保存淘宝联盟、京东联盟、抖音精选联盟官方API返回的数据。
- 每日数据：`scripts/generate_daily_data.py` 生成 `data/recommendations.json`，并写入 `docs/daily/YYYY-MM-DD.md` 归档。
- 云端运行：GitHub Actions `.github/workflows/daily-update.yml` 每天北京时间10:00自动运行，电脑关机不影响。

## 每日流程
1. 10:00 GitHub Actions 云端启动。
2. 依次刷新淘宝联盟、京东联盟、抖音精选联盟；单个平台失败不影响其他平台。
3. 只接受品牌及全部规格词精确匹配的SKU，并标记24小时新鲜、24-72小时老化、72小时以上过期。
4. 读取商品池，按评分模型重算并选出10个候选，前3个作为当天最建议上架。
5. 更新站点数据并部署到 GitHub Pages，写入每日归档。

## 评分规则
总分为1-10分：

`总分 = 跨平台热度30% + 价格竞争力20% + 利润可行性20% + 官方30天销量10% + 复购率10% + 差异化空间5% + 上架可操作性5% - 风险扣分`

- 跨平台热度：淘宝/天猫、京东、抖音、小红书的榜单、热词、报告、种草内容。
- 价格竞争力：只使用72小时内官方API精确SKU的预估到手价。
- 利润可行性：按 `建议到手价 * 78%` 倒推成本上限，预留15%-25%毛利和约7%平台/优惠/退货损耗。
- 官方30天销量：只使用联盟接口明确返回的30天销量；评价、榜单和内容互动不代替销量。
- 上架可操作性：品牌经销现货、组合装、赠品、规格差异化能力。
- 风险扣分：控价、重货运费、液体破损、功效合规、品控投诉。

## 复盘节奏
- 72小时：观察曝光、点击、收藏加购；无曝光先改标题和首图。
- 7天：看转化、退款和实际毛利，淘汰低效品。
- 14天：保留10-20个稳定SKU形成小货盘。
- 30天：沉淀主推品、利润品、活动品。
